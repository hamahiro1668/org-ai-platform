## 2. 全体像：何が変わったか

### 2.1 LLM アーキテクチャ

| 以前（イメージ） | 変更後 |
|------------------|--------|
| STARTER → Groq、PRO → OpenAI、MAX → Anthropic（フォールバック Groq） | **全プラン Anthropic Claude のみ** |
| ストリーミングは Groq のみ | **ストリーミングも Anthropic** |
| フロントのタスクマネージャーが Groq を直接叩ける構成あり | **常に API Gateway `/api/llm/chat` → ai-engine** |

プランによる差別化は **モデル品質（Haiku / Sonnet / Opus）** と **共有型で定義した月間コール上限**に寄せている。

### 2.2 n8n と AI Engine の関係

チャット・タスク実行・ドキュメント系ワークフローは、可能な限り **`POST {AI_ENGINE_URL}/llm/chat`** に寄せ、`messages` / `department` / `org_id` / `plan` / `json_mode` をボディで渡す形に統一。  
これにより **OpenAI ノードや Groq 直叩きをワークフローから排除**し、キーとガバナンス（PII 等）を ai-engine に集約できる。

## 3. AI Engine（`apps/ai-engine`）

### 3.1 依存関係（`requirements.txt`）

- **`groq` を削除**し **`anthropic>=0.40.0`** に変更。

### 3.2 削除されたファイル

- `app/llm/providers/groq.py`
- `app/llm/providers/openai_provider.py`

### 3.3 `app/llm/router.py`

- **単一プロバイダ**: `AnthropicProvider` のみ。
- **`_PLAN_MODEL`**: DB のプラン文字列 → モデル ID  
  - `STARTER` → `MODEL_HAIKU`（`claude-haiku-4-5-20251001`）  
  - `PRO` → `MODEL_SONNET`（`claude-sonnet-4-6`）  
  - `MAX` → `MODEL_OPUS`（`claude-opus-4-7`）  
  未知プランは Haiku にフォールバック。
- **`_screen_user_messages`**: ユーザー発話のみ PII `screen()` → `chat` / `chat_stream` の両方で共用。
- **`chat`**: `_resolve_model(plan)` でモデル選択、`provider.chat(..., json_mode=json_mode)`。
- **`chat_stream`**: コメントどおり Anthropic の `messages.stream` でトークンを yield（Groq 専用だった経路を置換）。

### 3.4 `app/llm/providers/anthropic_provider.py`

- モデル定数: `MODEL_HAIKU` / `MODEL_SONNET` / `MODEL_OPUS`、`DEFAULT_MODEL` は Sonnet。
- **`_split_system_and_messages`**: `system` ロールを連結して Messages API の `system` に、それ以外を `messages` に。
- **`json_mode`**: `system` に「JSON のみ・コードフェンス禁止」のヒントを追記（`JSON_MODE_HINT`）。
- **`chat`**: tenacity によるリトライは維持。**モデルフォールバック（Sonnet → Haiku）は削除**し、例外時はそのまま伝播。
- 応答テキスト: `response.content` を走査し **`type == "text"` のブロックのみ連結**（tool_use 等は無視）。
- **`chat_stream`（新規）**: `AsyncAnthropic.messages.stream` の `text_stream` を yield。

### 3.5 `app/main.py`

- **`GET /health`**: `service: ai-engine` を追加。
- **`GET /ready`（新規）**: `ANTHROPIC_API_KEY` の有無で `status` が `ok` / `degraded`、`anthropic: boolean` を返す。
- **`/orchestrate` のエラー判定**: メッセージに含まれるキー名を **`GROQ_API_KEY` → `ANTHROPIC_API_KEY`** に変更。
- **`orchestrate_stream` のログ**: `provider` を `groq` → **`anthropic`**、`model` を **`_resolve_model(request.plan)`** に。

### 3.6 `app/orchestrator/orchestrator.py`

- `log_llm_call` の **`provider` を `"groq"` → `"anthropic"`**（実モデル名は `response.model`）。

## 7. n8n ワークフロー（`apps/n8n-workflows`）

### 7.1 削除されたエクスポート JSON

| ファイル | おおよその役割（旧） |
|----------|----------------------|
| `task-email-send.json` | メール送信系サブフロー |
| `task-sns-post.json` | SNS 投稿系 |
| `sns-multi-platform.json` | マルチプラットフォーム SNS |

※ 実運用では Render の n8n にインポート済みワーカーを **削除・差し替えするか**は別途確認が必要。

### 7.2 `chat-response.json`（変更後の流れ）

1. Execute Workflow Trigger  
2. **Build Prompt**: 部署別システムプロンプト、`history` 最大 8 件、`plan` / `aiEngineUrl` を runData から引き継ぎ  
3. **AI Engine Chat**: `POST {{aiEngineUrl}}/llm/chat`、`json_mode: false`  
4. **Format Response**: 応答の `content` と部署を整形

Groq/OpenAI ノードはなし。**チャット応答も Claude（プラン連動）は ai-engine に集約**。

### 7.3 `task-execute.json`

_linear なパイプライン_:

`Log Start` → **Detect Task Type**（入力・`taskType` ヒント・部署から `taskMode` と `plan` / `aiEngineUrl`）→ **Build Structured Prompt**（モード別 JSON 生成プロンプト、`general` はプレーンテキスト）→ `Log Task Mode` → **AI Engine Chat**（`json_mode` はモード依存）→ `Log Done` → **`Complete Task` webhook**（`output` に `content`）

Gateway から渡る **`taskType` / `plan` / `aiEngineUrl`** をコードノードが参照。

### 7.4 `task-document.json`

- プロンプトコメントを **GPT-4o → Claude（ai-engine 経由）** に変更。
- ノード **`GPT-4o Generate`（LangChain OpenAI + credentials）を削除**。
- **`HTTP Request → {aiEngineUrl}/llm/chat`**、`json_mode: true`。
- **Parse Output**: `LLMResponse.content` を `JSON.parse`（失敗時はエラーフォールバック構造）。
